Welcome to the repository for Brainy, a web-based educational application created to support children between the ages of eight and ten who experience dyslexia. We developed this Minimum Viable Product during an intensive two-week university project with the goal of turning common reading and spelling frustrations into an engaging, stress-free experience. The application focuses on tackling specific literacy challenges, such as mixing up letter sequences, confusing similar-looking letters, and struggling with homophones (words that sound the same but are written differently). To achieve this, we built three interactive mini-games where users can construct words from scrambled letters, choose words based on contextual meaning, and practice differentiating homophones.

The entire platform was built using standard web technologies, including HTML, CSS, and vanilla JavaScript, which makes it incredibly lightweight and easy to run directly in any web browser without needing a complex backend setup. Because accessibility was our top priority, we deliberately designed the user interface to be as clean and distraction-free as possible. We integrated the OpenDyslexic font to improve readability and utilized soft, high-contrast color palettes to help the text stand out clearly. 

This was a group assignment for our university. This project was a great opportunity for us to combine our skills in UI/UX design and frontend development. It also gave us practical, hands-on experience in coordinating our workflow and managing our codebase as a team using Git for version control. If you would like to run the application locally, all you need to do is clone this repository and open the main HTML file - index.html, in your browser. We are really proud of what we managed to build within such a tight timeframe and hope it serves as a helpful, accessible tool for early learners.
